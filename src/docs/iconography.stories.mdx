# Iconography

* Icons will be SVG format.
* SVGs will use the current text color as the fill.
* Some icons will remain the same size; others will scale for different screen sizes. See individual components.

## How to configure

Define classes in **src/assets/util.styles.scss**.

<Source language='scss' code={dedent`
  /* SVG */ 
  svg {
    @apply fill-current;	     /* fill: currentColor; */
  }
  .icon {
    @apply w-full;             /* width: 100%; */
  }
`} />

## Accessibility considerations

Relevant [WCAG 2.1 guidelines](https://www.w3.org/WAI/WCAG21/quickref/):

* Same guidelines as for <a href="/?path=/story/foundation-color--page#accessibility-considerations">Color</a>.

* [1.1.1 Non-text Content (Level A)](https://www.w3.org/WAI/WCAG21/quickref/#non-text-content)<br />
  All non-text content that is presented to the user has a text alternative that serves the equivalent purpose. 
* [1.3.6 Identify Purpose (Level AAA)](https://www.w3.org/WAI/WCAG21/quickref/#identify-purpose)<br />
  In content implemented using markup languages, the purpose of User Interface Components, icons, and regions can be programmatically determined. 
* [2.5.5 Target Size (Level AAA)](https://www.w3.org/WAI/WCAG21/quickref/#target-size)<br />
  The size of the target for pointer inputs is at least 44 by 44 CSS pixels except when: 
  * Equivalent: The target is available through an equivalent link or control on the same page that is at least 44 by 44 CSS pixels;
  * Inline: The target is in a sentence or block of text;
  * User Agent Control: The size of the target is determined by the user agent and is not modified by the author;
  * Essential: A particular presentation of the target is essential to the information being conveyed.
