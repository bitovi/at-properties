# Iconography

* Icons remain the same size; they are *not* scaled for different screen sizes.
* Icons will be SVG format.
* SVGs will use the current text color as the fill.

## How to configure

Define classes in **src/assets/util.styles.scss**.

<Source language='css' code={dedent`
  /* SVG */ 
  svg {
    @apply fill-current;	     /* fill: currentColor; */
  }
`} />

## Social

Social icons will be 32 x 32px, with 24px of space between icons, and 32px of space above the row of icons. Note the `:hover` state: the icon fill color changes to the social brand color.

Social "icons" are actually logos and must follow third-party brand guidelines like any other logo. Guidelines define not only the logo itself, but also it's color, size, placement, and spacing. Please reference each brand's fair use guidelines.

* **YouTube** - [https://www.youtube.com/about/brand-resources/#logos-icons-colors](https://www.youtube.com/about/brand-resources/#logos-icons-colors)

## Accessibility considerations

Relevant [WCAG 2.1 guidelines](https://www.w3.org/WAI/WCAG21/quickref/):

* Same guidelines as for <a href="/?path=/story/foundation-color--page#accessibility-considerations">Color</a>.
* All non-text content that is presented to the user has a text alternative that serves the equivalent purpose. [1.1.1 Non-text Content (Level A)](https://www.w3.org/WAI/WCAG21/quickref/#non-text-content)
* In content implemented using markup languages, the purpose of User Interface Components, icons, and regions can be programmatically determined. [1.3.6 Identify Purpose (Level AAA)](https://www.w3.org/WAI/WCAG21/quickref/#identify-purpose)
